À propos

Je ne suis pas développeur de formation. Je suis un profil avant-vente senior — 20 ans à expliquer des produits complexes, qualifier des deals, aligner des équipes techniques et des décideurs business.

Ce qui a changé : j'ai appris à livrer des produits complets avec l'IA. Pas des prototypes. Des applications en production, utilisées par de vraies personnes, avec auth, persistance, et patterns AI avancés.

Ma conviction : l'AI-native builder n'est pas celui qui sait coder le mieux — c'est celui qui sait livrer. Comprendre ce qui se passe sous le capot. Piloter l'IA comme un vrai collaborateur technique. Itérer vite. Documenter proprement.

Stack & patterns AI

6 patterns AI implémentés en production. Pas de la théorie — des apps réelles qui tournent.

🧠

RAG multi-source

Profil candidat + CV + docs de référence injectés en system prompt. Pas de vector DB — full context pour usage mono-utilisateur.

📐

Structured output

JSON schema strict validé côté client. Chaque agent retourne un schema défini qui pilote l'affichage UI.

🔧

Tool-calling

Job Scanner : texte/URL → Claude parse → JSON structuré. Classification roleType basée sur les responsabilités réelles.

⛓️

Multi-agent pipeline

7 agents avec état partagé. Scan → Qualify → ATS → Adapt → Outreach. Même job + contexte RAG passé entre agents.

✨

Few-shot engineering

Outreach Generator : 2 variantes, ton pair-à-pair, liste de phrases interdites, bilingue FR/EN.

🧹

Output cleaning

Pipeline cleanAIText() — supprime les artefacts markdown avant stockage pour des outputs professionnels.
